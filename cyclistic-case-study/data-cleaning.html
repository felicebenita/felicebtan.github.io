Data Cleaning Process

1. Document Description

For the purpose of this case study, only data collected between September 2021 - August 2022 will be assessed.

Please note that Cyclistic is a fictional company. Raw data has been collected by Motivate International Inc, the company which operates the City of Chicago’s Divvy bicycle sharing service. The license to use this public dataset can be found here.

 

2. Data Tools Used

In this analysis, I will be using Microsoft Excel XLS for data sets fomat, MySQL for the data cleaning, analysis and manipulation of data, and Navicat Premium for the Database Graphical UI.

 

Excel

Load all .CSV format data sets and saved them into .XLS format.

 

MySQL

  1. Import and merge all 12 months trip data into one table (divvy_tripdata).

Total records of divvy_tripdata : 5883043

  1. Change data type of field started_at and ended_at from string type to datetime type to help with analysis.
  2. Create a column called “day_of_week” with smallint data type that calculate the day of the week that each ride started.

Query syntax :

UPDATE divvy_tripdata SET day_of_week = WEEKDAY(started_at);

Affected rows : 5883043

Total records of divvy_tripdata : 5883043

  1. Create a column called “ride_length” with integer data type that calculate the length of each ride by subtracting the column “started_at” from the column “ended_at” in seconds.

Query syntax :

UPDATE divvy_tripdata SET ride_length = TIMESTAMPDIFF(SECOND, started_at, ended_at);

Affected rows : 5883043

Total records of divvy_tripdata : 5883043

  1. Remove rows with ride_length < 0

There were a few invalid data points with ride lengths less than 0. These data points should be deleted from the cleaned dataset.

These deleted 135 records of data from total records 5883043 into 5882908.

Query syntax :

DELETE FROM divvy_tripdata WHERE ride_length < 0;

Affected rows : 135

Total records of divvy_tripdata : 5882908

  1. Remove incomplete rows

There were a few instances where there were no recorded station names. These incomplete rows of data should be removed.

6.1 Delete rows with start_station_name is null or empty

These deleted 884338 records of data from total records 5882908 into 4998570.

Query syntax :

DELETE FROM divvy_tripdata WHERE start_station_name IS NULL

OR start_station_name = '';

Affected rows : 884338

Total records of divvy_tripdata : 4998570

6.2 Delete rows with end_station_name is null or empty

These deleted 438523 records of data from total records 4998570 into 4560047.

Query syntax :

DELETE FROM divvy_tripdata WHERE end_station_name IS NULL

OR end_station_name = '';

Affected rows : 438523

Total records of divvy_tripdata : 4560047

  1. Remove duplicates

The ride_id column is unique to each ride. This column should be reviewed to see if there are any duplicates to delete.

Checking :

SELECT ride_id, COUNT(ride_id) FROM divvy_tripdata

GROUP BY ride_id

HAVING COUNT(ride_id) > 1;

Total records : 0

  1. Check value for member_casual if there is unvalid data other than member or casual

Query syntax :

SELECT DISTINCT member_casual FROM divvy_tripdata;

The above line of code returned two user types used in the cleaned dataset : member & casual.

  1. Check rideable type

Query syntax :

SELECT DISTINCT rideable_type FROM divvy_tripdata;

The above line of code returned three bike types used in the cleaned dataset.

  1. Check station name

Firstly, a data table which lists the unique station names should be created.

Query syntax :

INSERT INTO divvy_stations (id, name, total_trip, city, latitude, longitude, dpcapacity, online_date)

SELECT DISTINCT start_station_id, start_station_name, COUNT(ride_id), 'Chicago', 0, 0, 0, NOW()

FROM divvy_tripdata

GROUP BY start_station_id, start_station_name

ORDER BY start_station_name;

Affected rows : 1382

Total records of divvy_stations : 1382

The above line of code inserted 1382 records of unique station_name used in the cleaned dataset and the total number of trips for each station_name. This identify how many times station_name used in the cleaned dataset.

If we filter the total_trip of the station_name data table by count < 12 it highlights that there are a few stations which have been added and/or removed from Cyclistic’s portfolio as well as help identify which stations were not used in a particular month.
