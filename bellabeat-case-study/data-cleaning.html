Data Cleaning Process

1. Document Description

Data was obtained on Kaggle from the owner Mobius. The data was collected through a distributed survey through Amazon Mechanical Turk. 33 fitbit users consented to the submission of personal tracker data, including minute-level output for physical activity, heart rate, and sleep monitoring from 04/12/2016 to 05/12/2016.

 

2. Data Tools Used

In this analysis, I will be using Microsoft Excel XLS for data sets fomat and basic cleaning, MySQL for the data cleaning, analysis and manipulation of data, Navicat Premium for the Database Graphical UI, and Tableau for the creation of visuals.

 

Excel

  1. Load all .CSV format data sets and saved them into .XLS format.
  2. Datasets were checked for duplicates using the "Remove Duplicates" data tool on all columns. The "sleepDay_merged" file contained 3 duplicates that were removed. None of the other hourly or daily datasets contained duplicates.

  5. For the "dailyActivity_merged" data table, query checking was used to identify number of days the data was tracked for each user (not exceed 31 days).

Query syntax :

SELECT Id, COUNT(*) FROM dailyactivity_merged

GROUP BY Id

HAVING COUNT(*) > 31;

The above line of code returned 0 data user that number of days the data was tracked exceed 31 days.

  1. For the "dailyActivity_merged" data table, query checking was used to identify total time tracked a day for each user to verify that no times exceeded the total time in a day (not exceed 1440 minutes).

Query syntax :

SELECT (VeryActiveMinutes + FairlyActiveMinutes + LightlyActiveMinutes +

SedentaryMinutes) AS TotalMinutes

FROM dailyactivity_merged

WHERE (VeryActiveMinutes + FairlyActiveMinutes + LightlyActiveMinutes +

SedentaryMinutes) > 1440;

The above line of code returned 0 data user that total time tracked a day was exceeded the total time in a day (exceed 1440 minutes).

 

Manipulation of Data

 

  1. Create new table named “dailyandsleep_merged” to merge dailyActivity with SleepDay records using Left Join clause.

Query syntax :

INSERT INTO dailyandsleep_merged (Id, ActivityDate, TotalSteps, TotalDistance,

TrackerDistance, LoggedActivitiesDistance, VeryActiveDistance,

ModeratelyActiveDistance, LightActiveDistance, SedentaryActiveDistance,

VeryActiveMinutes, FairlyActiveMinutes,LightlyActiveMinutes, SedentaryMinutes,

Calories, SleepDay, TotalSleepRecords, TotalMinutesAsleep, TotalTimeInBed)

SELECT a.Id, a.ActivityDate, a.TotalSteps, a.TotalDistance, a.TrackerDistance,

a.LoggedActivitiesDistance, a.VeryActiveDistance, a.ModeratelyActiveDistance,

a.LightActiveDistance, a.SedentaryActiveDistance, a.VeryActiveMinutes,

a.FairlyActiveMinutes, a.LightlyActiveMinutes, a.SedentaryMinutes, a.Calories,

b.SleepDay, b.TotalSleepRecords, b.TotalMinutesAsleep, b.TotalTimeInBed

FROM dailyactivity_merged a LEFT JOIN sleepday_merged b

ON a.Id = b.Id AND a.ActivityDate = b.SleepDay;

Affected rows : 940

Total records of dailyandsleep_merged : 940

  1. Create new table to save the summary statistics named “summarydaily_merged”.

Query syntax :

CREATE TABLE summarydaily_merged (

    Id varchar(255),

    AvgTotalSteps decimal(10,3),

    AvgVeryActiveMinutes decimal(10,3),

    AvgFairlyActiveMinutes decimal(10,3),

    AvgLightlyActiveMinutes decimal(10,3),

    AvgSedentaryMinutes decimal(10,3),

    AvgCalories decimal(10,3),

    AvgTotalMinutesAsleep decimal(10,3),

    AvgTotalTimeInBed decimal(10,3)

);
  1. Calculate average of total steps of each users and save it to table summarydaily_merged.

Query syntax :

INSERT INTO summarydaily_merged (Id, AvgTotalSteps)

SELECT Id, AVG(TotalSteps)

FROM dailyandsleep_merged

GROUP BY Id;

Affected rows : 33

Total records of summarydaily_merged : 33

  1. Calculate average of activity minutes by level of each users and save it to table summarydaily_merged.

Query syntax :

UPDATE summarydaily_merged s

AvgLightlyActiveMinutes = (SELECT AVG(LightlyActiveMinutes) FROM dailyandsleep_merged

AvgSedentaryMinutes = (SELECT AVG(SedentaryMinutes) FROM dailyandsleep_merged
